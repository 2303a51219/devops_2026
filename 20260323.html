Scenario Description

You are working as a Cloud Engineer in an organization that wants to provision cloud infrastructure using Infrastructure as Code (IaC). Your task is to use Terraform to create a Virtual Machine (VM) on a cloud platform of your choice — AWS, Azure, or GCP.

01
Setup
  • Install Terraform on your local system.
  • Create a cloud provider account (AWS / Azure / GCP) and obtain access credentials.
  • Configure the cloud provider in Terraform using environment variables or config files.
02
Terraform Configuration Files

Create the following file structure in your project directory:

📁 terraform-vm/
  📄 main.tf  → Provider and resource definitions
  📄 variables.tf  → Input variables
  📄 outputs.tf  → Output values (e.g. public IP)

Define a VM instance resource with the following attributes:

  • Instance name — unique identifier for the VM
  • Machine type — CPU/RAM size (e.g. t2.micro, e2-micro)
  • Region and availability zone — deployment location
main.tf — GCP Example 
# Configure GCP provider
provider "google" {
  project = var.project_id
  region  = var.region
}

# Create a VM instance
resource "google_compute_instance" "vm_instance" {
  name         = var.instance_name
  machine_type = var.machine_type
  zone         = var.zone

  boot_disk {
    initialize_params {
      image = "debian-cloud/debian-11"
    }
  }

  network_interface {
    network = "default"
    access_config {}
  }
}
variables.tf 
variable "project_id"    { default = "my-gcp-project" }
variable "region"        { default = "us-central1" }
variable "zone"          { default = "us-central1-a" }
variable "instance_name" { default = "lab21-vm" }
variable "machine_type"  { default = "e2-micro" }
outputs.tf 
output "vm_public_ip" {
  value = google_compute_instance.vm_instance.network_interface[0].access_config[0].nat_ip
}
03
Execution Tasks
Step 1 — Initialize
terraform init
Downloads provider plugins and sets up the working directory.
Step 2 — Plan
terraform plan
Previews infrastructure changes without applying them.
Step 3 — Apply
terraform apply
Provisions the actual cloud resources as defined.
04
Advanced Terraform Operations
  • Modify VM configuration — edit variables or resource blocks, then re-run terraform apply.
  • View Terraform state — inspect current infrastructure state using terraform show.
  • Destroy infrastructure — tear down all provisioned resources cleanly.
Terminal 
$ terraform show               # View current state
$ terraform state list          # List all resources
$ terraform destroy             # Destroy all resources
⭐
Bonus Challenge

Use Variables for Instance Size

Parameterize the machine type so the VM size can be changed at apply time without editing main.tf.

Output the Public IP Address

Use outputs.tf to expose the VM's public IP so it can be used downstream by other tools or scripts.
